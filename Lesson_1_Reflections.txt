Reflection 1: How did viewing a diff between two versions help you spot the bug?

    Cuando puedo ver las diferencias entre los dos archivos, es muy fácil de observar los cambios en el código. Como puedo ver las diferencias entre la vieja y la nueva versión del código, puedo detectar los errores fácilmente; basta con comparar las diferencias en las líneas de los dos códigos . Si la versión anterior del código estaba trabajando sin ningún problema, y si la nueva versión falla. Existe la posibilidad de que los cambios que he realizado en el código antiguo puedan ser el problema o el error en la nueva versión del código .
    
Reflection 2: How could having easy access to the entire history of a file make you a more efficient programmer in the long term?

    Tener acceso a las versiones anteriores de los programas que estoy desarrollando me permite corregir errores más fácilmente y de forma más rápida. Esto hace que sea un mejor programador ya que mejora mi rendimiento al momento de escribir código. También me permite volver a un punto en el que el desarrollo que llevo a cabo funciona sin problemas, permitiendome retomar ese código para hacer un desarrollo más eficientes y sin errores como los que pude haber cometido en el nuevo archivo por el cual tuve que buscar una vieja versión de mi desarrollo.

Reflection 3: What do you think are the pros and cons of manually choosing when to create a commit, like you do in Git, vs having versions automatically saved, like Google
docs does?

    Ventajas: Puedo guardar las versiones con cambios lógicos y significativos, puedo dar una descripción del cambio realizado en un commit especifico para saber que cambios fueron hechos hasta ese punto.

    Desventajas: Puede ser que olvide hacer un commit en algún cambio importante, puedo dejar pasar mucho tiempo sin hacer un commit sin darme cuenta.

Reflection 4: Why do you think some version control systems, like Git, allow saving multiple files in one commit, while others, like Google Docs, treat each file separately?

    Creo que sistemas de versiones como Git tienen claro que hay proyectos en los que varios archivos están relacionados entre si, y si el código de uno de esos archivos cambia, el código de los archivos relacionados con el probablemente también deba cambiar.

Reflection 5: How can you use the commands git log and git diff to view the history of files?

    Puedo usar git log para ver el historial de commits hechos con información del mensaje que dice lo que se hizo en ese commit y un registro de la fecha y hora en que se hizo el commit. El comando git diff compara dos commit para ver las lineas de código que se agregaron en la versión más reciente y se quitaron en la versión más antigua del código.

Reflection 6: How might using version control make you more confident to make changes that could break something?

    Ya que puedo guardar versiones del código que funcionan sin problema hasta cierto punto del desarrollo, puedo hacer cambios significativos y que podrían causar errores ya que en caso de que dichos cambios causen problemas, puedo volver a una versión anterior del código en donde este funcionara sin problema y no tuviera todos los cambios que causaron problemas.

Reflection 7: Now that you have your workspace set up, what do you want to try using Git for?

    Ya que tengo configurado Git en mi computador y además tengo una idea más clara de todo lo que puedo hacer con Git, lo quiero usar para el desarrollo del Sistema de Información que debo llevar a cabo en mi proyecto de grado de la universidad.
